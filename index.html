Claude Code vs Claude Chat

Two ways to work with Claude — one for conversations, one that lives inside your codebase. Understanding the difference changes how you build.

CLI · Agentic

Claude Code

$ claude  ·  Terminal-native AI agent

A command-line agent that runs directly in your terminal. It reads your files, understands your project, runs commands, edits code, and commits changes — all with your approval.

  • Reads, edits, and creates files in your local filesystem
  • Runs shell commands, tests, and build scripts
  • Native Git integration — diffs, commits, branches, PRs
  • Multi-file reasoning across entire codebases
  • Extensible with MCP servers and custom tools
  • CLAUDE.md project memory persists across sessions
  • Spawns parallel sub-agents for complex tasks

Web · Conversational

Claude Chat

claude.ai  ·  Browser-based assistant

A conversational interface in your browser. Ask anything — get analysis, writing, explanations, brainstorming, and code snippets. No setup, no local access required.

  • Instant access — no installation, runs in any browser
  • Upload files, images, and PDFs for analysis
  • Projects keep context organised across conversations
  • Write, edit, summarise, and translate documents
  • Brainstorm ideas, get feedback, explore questions
  • Voice input and rich text response formatting
  • Team collaboration with shared Projects
Claude — New conversation
Can you explain how async/await works in JS?
Async/await is syntactic sugar over Promises. An async function always returns a Promise, and await pauses execution until that Promise resolves — without blocking the thread.
What about error handling?
Wrap your await in a try/catch. Rejected Promises become catchable errors, just like synchronous exceptions…

When to reach for

Claude Code

  • → Refactoring a module across dozens of files
  • → Debugging a failing test suite end-to-end
  • → Scaffolding a new project from scratch
  • → Writing and running database migrations
  • → Automating repetitive dev workflows
  • → Reviewing diffs and creating pull requests
  • → Integrating APIs with full context of your stack
  • → Setting up CI/CD pipelines and config files

When to reach for

Claude Chat

  • → Getting a quick explanation of a concept
  • → Drafting or editing a document or email
  • → Brainstorming ideas without a codebase
  • → Summarising a PDF or long document
  • → Asking questions about an image or diagram
  • → Exploring a topic through dialogue
  • → Collaborating with teammates on shared context
  • → Learning something new with back-and-forth Q&A
